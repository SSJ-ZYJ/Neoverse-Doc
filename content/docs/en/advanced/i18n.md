---
title: i18n (Internationalization)
description: Neoverse-Doc's multilingual architecture and copy management
---

Neoverse-Doc builds on fumadocs' built-in i18n system, combining dictionary-driven UI copy with directory-based document content to deliver zero-hardcoding multilingual switching. Chinese and English are both officially enabled today.

## Architecture Overview

```text
src/
  dictionaries/
    zh.ts          ← Chinese language pack
    en.ts          ← English language pack
    index.ts       ← Dictionary aggregator + getDictionary(locale)
  lib/
    source.ts      ← loader configured with i18n: { defaultLanguage, languages, parser: 'dir' }
  app/
    page.tsx       ← Root client-side redirect to /{defaultLocale}
    [lang]/
      layout.tsx   ← RootProvider injects the dictionary + generateMetadata per locale
      page.tsx     ← Home page (lang comes from the route param)
      docs/
        layout.tsx ← DocsLayout, tree={source.pageTree[lang]}
        [...slug]/page.tsx
      guestbook/
        page.tsx
content/docs/
  zh/              ← Chinese document subtree
  en/              ← English document subtree
```

> Static export (`output: 'export'`) does not support Next.js middleware, so the language redirect at the root path is handled by the root `page.tsx` via a client-side `<meta http-equiv="refresh">` plus `window.location.replace`.

## Dictionary Structure

Each language pack (`src/dictionaries/{lang}.ts`) has two parts:

- **fumadocs UI overrides** — override the built-in copy from `defaultTranslations` for search, theme, navigation, and more
- **Application-specific copy** — project-specific strings such as home tagline, nav labels, and community section

```ts
import { defaultTranslations } from 'fumadocs-ui/i18n';

export const en = {
  ...defaultTranslations,
  displayName: 'English',
  search: 'Search',
  // ...
  // Application copy
  tagline: 'Next-Generation Tech Aesthetics Full-Stack Documentation Engine',
  communityTitle: 'Community',
} as const;
```

The aggregator `src/dictionaries/index.ts` exposes `Locale`, `locales`, `defaultLocale`, and the `getDictionary(locale)` helper.

## Usage

**Server components (pages / layouts)** — pull the dictionary based on the route's `lang` param

```tsx
import { getDictionary, type Locale, defaultLocale } from '@/dictionaries';

export default function Page({ params }: { params: { lang: Locale } }) {
  const dict = getDictionary(params.lang ?? defaultLocale);
  return <h1>{dict.communityTitle}</h1>;
}
```

**Client components** — access via fumadocs' `useTranslations()` hook

```tsx
'use client';
import { useTranslations } from 'fumadocs-ui/contexts/i18n';

export function MyComponent() {
  const t = useTranslations();
  return <span>{t.communityTitle}</span>;
}
```

## Multilingual Document Content

fumadocs-core's loader enables directory-based parsing with `parser: 'dir'`, organizing docs by language sub-directories:

```text
content/docs/
  zh/
    getting-started/
    advanced/
  en/
    getting-started/
    advanced/
```

Each language maintains its own `meta.json` for page titles and ordering:

```json
// content/docs/en/getting-started/meta.json
{ "title": "Getting Started", "pages": ["index", "syntax-example"] }
```

Page components fetch the localized page via `source.getPage(slug, locale)`:

```tsx
const page = source.getPage(slug, locale);
```

The sidebar tree (`source.pageTree`) is a `Record<lang, Root>`, so index by `lang`:

```tsx
<DocsLayout tree={source.pageTree[lang]} {...options} />
```

## Language Switcher

Once the loader detects an `i18n` config, fumadocs-ui automatically renders the `languageSelect` slot in the upper-right of the navigation bar — no manual UI wiring required. To customize the dropdown label, tweak the `displayName` field of each language pack.

To customize the switcher's appearance, override the default implementation in `DocsLayout`'s `slots.languageSelect` (see the `BaseSlots.languageSelect` type in fumadocs-ui).

## Adding a New Language

1. Create `src/dictionaries/{lang}.ts`, copying an existing language pack and translating every value (keep the keys identical).
2. Register the new language in the `dictionaries` and `locales` arrays inside `src/dictionaries/index.ts`.
3. Append the new locale to `i18n.languages` in both `source.config.ts` and `src/lib/source.ts`.
4. Add the corresponding language directory under `content/docs/`, then translate the Markdown files and `meta.json`.
5. If you want the Giscus comments to render in the new language, update the `GISCUS_LANG_MAP` in `src/components/guestbook.tsx`.
