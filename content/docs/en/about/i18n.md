---
title: i18n Internationalization
description: Multi-language architecture and copy management for Neoverse-Doc
---

Neoverse-Doc leverages fumadocs' built-in i18n system, combined with dictionary-based UI copy and directory-structured documentation content, enabling multi-language switching with zero hardcoded strings. Chinese and English interfaces and documentation are currently fully supported.

## Core Configuration

### i18n Definition

```typescript
// src/lib/i18n.ts
import { defineI18n } from 'fumadocs-core/i18n';

export const i18n = defineI18n({
  defaultLanguage: 'zh',
  languages: ['zh', 'en'],
  parser: 'dir',
});

export type Locale = (typeof i18n.languages)[number];
```

`parser: 'dir'` enables directory-based parsing, automatically organizing languages by `content/docs/{lang}/` subdirectories.

## Architecture Overview

```text
src/
├── dictionaries/
│   ├── index.ts       ← Dictionary aggregation + getDictionary(locale)
│   ├── zh.ts         ← Chinese language pack
│   └── en.ts         ← English language pack
├── lib/
│   ├── i18n.ts       ← Centralized i18n config (single source of truth)
│   ├── source.ts      ← loader with i18n integration
│   └── layout.shared.tsx ← fumadocs UI translations
└── app/
    ├── not-found.tsx ← Root localized 404 fallback
    └── [lang]/
        ├── layout.tsx ← RootProvider injects dictionary
        ├── not-found.tsx ← Locale-level localized 404 fallback
        └── docs/
            ├── not-found.tsx ← Docs-area 404 fallback
            └── [...slug]/
                ├── not-found.tsx ← Missing document fallback
                └── page.tsx
content/docs/
├── zh/              ← Chinese documents
└── en/              ← English documents
```

`src/components/localized-not-found.tsx` owns the shared 404 fallback UI. It reads copy from `src/dictionaries/` using the current route's `lang` segment, and falls back to `i18n.defaultLanguage` when no language segment exists.

### Icon Mapping

The icon handler in `source.ts` maps `icon: Name` strings from frontmatter / meta.json to lucide-react components per the official Fumadocs convention:

```typescript
// src/lib/source.ts
icon(icon) {
  if (!icon) return;
  if (icon in icons) {
    return createElement(icons[icon as keyof typeof icons]);
  }
}
```

## Language Pack Structure

Each language pack (`src/dictionaries/{lang}.ts`) consists of two parts:

- **fumadocs UI Overrides**: Override built-in copy such as search, theme, and navigation in `defaultTranslations`
- **Application Custom Copy**: Project-specific copy like homepage taglines, navigation labels, and community titles

```typescript
// src/dictionaries/zh.ts
import { defaultTranslations } from 'fumadocs-ui/i18n';

export const zh = {
  ...defaultTranslations,
  displayName: '中文',
  search: '搜索',
  // fumadocs UI overrides
  theme: '主题',
  // Application custom copy
  tagline: '下一代科技美学全栈文档引擎',
  communityTitle: '社区探讨',
} as const;
```

The aggregation layer `src/dictionaries/index.ts` exposes `Locale`, `locales`, `defaultLocale`, and the `getDictionary(locale)` utility function.

## Usage

### Server Components

Retrieve dictionary based on route `lang` parameter:

```tsx
import { getDictionary, type Locale, defaultLocale } from '@/dictionaries';

export default async function Page({
  params,
}: {
  params: Promise<{ lang: Locale }>;
}) {
  const { lang } = await params;
  const dict = getDictionary(lang ?? defaultLocale);
  return <h1>{dict.communityTitle}</h1>;
}
```

> Next.js 15+ requires `Promise` type for params.

### Client Components

Access via fumadocs' `useTranslations()` Hook:

```tsx
'use client';
import { useTranslations } from 'fumadocs-ui/contexts/i18n';

export function MyComponent() {
  const t = useTranslations();
  return <span>{t('communityTitle')}</span>;
}
```

## Multi-Language Content Documents

fumadocs-core's loader enables directory-based parsing via `parser: 'dir'`, organizing documents by language subdirectory:

```text
content/docs/
  zh/
    ch0/
    about/
    contributing/
  en/
    ch0/
    about/
    contributing/
```

Each language's `meta.json` independently maintains page titles and ordering:

```json
// content/docs/zh/about/meta.json
{
  "title": "About",
  "pages": ["index", "i18n", "theming", "search"],
  "root": true
}
```

Page components retrieve pages by language via `source.getPage(slug, locale)`:

```tsx
const page = source.getPage(slug, locale);
```

The sidebar tree (`source.pageTree`) is `Record<lang, Root>` and must be accessed by lang:

```tsx
<DocsLayout tree={source.pageTree[lang]} {...options} />
```

## Language Switcher

fumadocs-ui automatically renders the `languageSelect` slot (located at the top-right of the navbar) when it detects that the loader has `i18n` configured — no manual UI integration is needed. To customize the display text of dropdown items, adjust the `displayName` in each language pack.

To customize the switcher appearance, override the default implementation in `DocsLayout`'s `slots.languageSelect`.

## Adding a New Language

1. Append the new language to the `languages` array in `src/lib/i18n.ts`
2. Create `{lang}.ts` under `src/dictionaries/`, copy an existing language pack, and translate all values (keeping keys consistent)
3. Register the new language in `dictionaries` and `locales` within `src/dictionaries/index.ts`
4. Create the corresponding language directory under `content/docs/` and translate Markdown and `meta.json` by section

## Static Export

Since the project uses `output: 'export'` for static export, middleware is not supported. The root path `/` language redirect is handled by the root `page.tsx` via client-side redirect:

```tsx
// src/app/page.tsx
'use client';
// Redirect to /{defaultLocale} via window.location.replace
```

> For more fumadocs UI translation configuration, refer to `src/lib/layout.shared.tsx`.
