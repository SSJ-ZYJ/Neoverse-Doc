---
title: i18n Internationalization
description: Multi-language architecture and copy management for Neoverse-Doc
---

Neoverse-Doc leverages fumadocs' built-in i18n system, combined with dictionary-based UI copy and directory-structured documentation content, enabling multi-language switching with zero hardcoded strings. Chinese and English interfaces and documentation are currently fully supported.

## Architecture Overview

```text
src/
  dictionaries/
    zh.ts          ← Chinese language pack
    en.ts          ← English language pack
    index.ts       ← Dictionary aggregation + getDictionary(locale)
  lib/
    source.ts      ← loader with i18n: { defaultLanguage, languages, parser: 'dir' }
  app/
    page.tsx       ← Root path / client-side redirect to /{defaultLocale}
    [lang]/
      layout.tsx   ← RootProvider injects dictionary + generateMetadata per locale
      page.tsx     ← Home page (lang from route params)
      docs/
        layout.tsx ← DocsLayout, tree={source.pageTree[lang]}
        [...slug]/page.tsx
      guestbook/
        page.tsx
content/docs/
  zh/              ← Chinese document subtree
  en/              ← English document subtree
```

> Static export (`output: 'export'`) does not support Next.js middleware, so the root path language redirect is handled by the root `page.tsx` via client-side `<meta http-equiv="refresh">` + `window.location.replace`.

## Language Pack Structure

Each language pack (`src/dictionaries/{lang}.ts`) consists of two parts:

- **fumadocs UI Overrides**: Override built-in copy such as search, theme, and navigation in `defaultTranslations`
- **Application Custom Copy**: Project-specific copy like homepage taglines, navigation labels, and community titles

```ts
import { defaultTranslations } from 'fumadocs-ui/i18n';

export const zh = {
  ...defaultTranslations,
  displayName: '中文',
  search: '搜索',
  // ...
  // Application copy
  tagline: '下一代科技美学全栈文档引擎',
  communityTitle: '社区探讨',
} as const;
```

The aggregation layer `src/dictionaries/index.ts` exposes `Locale`, `locales`, `defaultLocale`, and the `getDictionary(locale)` utility function.

## Usage

**Server Components (Pages / Layouts)**: Retrieve dictionary based on route `lang` parameter

```tsx
import { getDictionary, type Locale, defaultLocale } from '@/dictionaries';

export default function Page({ params }: { params: { lang: Locale } }) {
  const dict = getDictionary(params.lang ?? defaultLocale);
  return <h1>{dict.communityTitle}</h1>;
}
```

**Client Components**: Access via fumadocs' `useTranslations()` Hook

```tsx
'use client';
import { useTranslations } from 'fumadocs-ui/contexts/i18n';

export function MyComponent() {
  const t = useTranslations();
  return <span>{t.communityTitle}</span>;
}
```

## Multi-Language Content Documents

fumadocs-core's loader enables directory-based parsing via `parser: 'dir'`, organizing documents by language subdirectory:

```text
content/docs/
  zh/
    ch0/
    about/
    contributing/
  en/
    ch0/
    about/
    contributing/
```

Each language's `meta.json` independently maintains page titles and ordering:

```json
// content/docs/en/contributing/meta.json
{ "title": "Contributing", "pages": ["index", "guide", "syntax-example"] }
```

Page components retrieve pages by language via `source.getPage(slug, locale)`:

```tsx
const page = source.getPage(slug, locale);
```

The sidebar tree (`source.pageTree`) is `Record<lang, Root>` and must be accessed by lang:

```tsx
<DocsLayout tree={source.pageTree[lang]} {...options} />
```

## Language Switcher

fumadocs-ui automatically renders the `languageSelect` slot (located at the top-right of the navbar) when it detects that the loader has `i18n` configured — no manual UI integration is needed. To customize the display text of dropdown items, adjust the `displayName` in each language pack.

To customize the switcher appearance, override the default implementation in `DocsLayout`'s `slots.languageSelect` (refer to the fumadocs-ui source `BaseSlots.languageSelect` type).

## Adding a New Language

1. Create `{lang}.ts` under `src/dictionaries/`, copy an existing language pack, and translate all values (keeping keys consistent)
2. Register the new language in `dictionaries` and `locales` within `src/dictionaries/index.ts`
3. Append the new locale to `i18n.languages` in both `source.config.ts` and `src/lib/source.ts`
4. Create the corresponding language directory under `content/docs/` and translate Markdown and `meta.json` by section
5. To use the corresponding language in Giscus comments, update `src/components/guestbook.tsx`'s `GISCUS_LANG_MAP`
