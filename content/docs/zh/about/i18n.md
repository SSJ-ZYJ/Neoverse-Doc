title: i18n 国际化

Neoverse-Doc 基于 fumadocs 内建 i18n 体系，搭配字典化 UI 文案与目录化文档内容，零硬编码即可支持多语言切换。当前已正式启用中文与英文两套界面与文档。

## 核心配置

### i18n 定义

```typescript
// src/lib/i18n.ts
import { defineI18n } from 'fumadocs-core/i18n';

export const i18n = defineI18n({
  defaultLanguage: 'zh',
  languages: ['zh', 'en'],
  parser: 'dir',
});

export type Locale = (typeof i18n.languages)[number];
```

`parser: 'dir'` 启用目录化解析，按 `content/docs/{lang}/` 子目录自动划分语言。

## 架构概览

```text
src/
├── dictionaries/
│   ├── index.ts       ← 字典聚合 + getDictionary(locale)
│   ├── zh.ts         ← 中文语言包
│   └── en.ts         ← 英文语言包
├── lib/
│   ├── i18n.ts       ← 集中式 i18n 配置（唯一来源）
│   ├── source.ts      ← loader 接入 i18n 配置
│   └── layout.shared.tsx ← fumadocs UI 翻译
└── app/
    └── [lang]/
        ├── layout.tsx ← RootProvider 注入字典
        └── docs/
            └── [...slug]/page.tsx
content/docs/
├── zh/              ← 中文文档
└── en/              ← 英文文档
```

### 图标映射

`source.ts` 中的 icon 处理器按 Fumadocs 官方约定，将 frontmatter / meta.json 里的 `icon: Name` 字符串映射为 lucide-react 组件：

```typescript
// src/lib/source.ts
icon(icon) {
  if (!icon) return;
  if (icon in icons) {
    return createElement(icons[icon as keyof typeof icons]);
  }
}
```

## 语言包结构

每个语言包 (`src/dictionaries/{lang}.ts`) 由两部分组成：

- **fumadocs UI 覆盖**：覆盖 `defaultTranslations` 中的搜索、主题、导航等内置文案
- **应用自定义文案**：首页 tagline、导航标签、社区探讨等项目特有文案

```typescript
// src/dictionaries/zh.ts
import { defaultTranslations } from 'fumadocs-ui/i18n';

export const zh = {
  ...defaultTranslations,
  displayName: '中文',
  search: '搜索',
  // fumadocs UI 覆盖
  theme: '主题',
  // 应用自定义文案
  tagline: '下一代科技美学全栈文档引擎',
  communityTitle: '社区探讨',
} as const;
```

聚合层 `src/dictionaries/index.ts` 暴露 `Locale`、`locales`、`defaultLocale` 与 `getDictionary(locale)` 工具函数。

## 使用方式

### 服务端组件

根据路由 `lang` 参数取字典：

```tsx
import { getDictionary, type Locale, defaultLocale } from '@/dictionaries';

export default async function Page({
  params,
}: {
  params: Promise<{ lang: Locale }>;
}) {
  const { lang } = await params;
  const dict = getDictionary(lang ?? defaultLocale);
  return <h1>{dict.communityTitle}</h1>;
}
```

> Next.js 15+ 的 params 需要使用 `Promise` 类型。

### 客户端组件

通过 fumadocs 的 `useTranslations()` Hook 访问：

```tsx
'use client';
import { useTranslations } from 'fumadocs-ui/contexts/i18n';

export function MyComponent() {
  const t = useTranslations();
  return <span>{t('communityTitle')}</span>;
}
```

## 内容文档的多语言

fumadocs-core 的 loader 通过 `parser: 'dir'` 启用目录化解析，按语言子目录组织文档：

```text
content/docs/
  zh/
    ch0/
    about/
    contributing/
  en/
    ch0/
    about/
    contributing/
```

每种语言的 `meta.json` 独立维护页面标题与排序：

```json
// content/docs/zh/about/meta.json
{
  "title": "关于",
  "pages": ["index", "i18n", "theming", "search"],
  "root": true
}
```

页面组件通过 `source.getPage(slug, locale)` 按语言取页面：

```tsx
const page = source.getPage(slug, locale);
```

侧栏树 (`source.pageTree`) 为 `Record<lang, Root>`，需按 lang 取：

```tsx
<DocsLayout tree={source.pageTree[lang]} {...options} />
```

## 语言切换器

fumadocs-ui 在检测到 loader 配置了 `i18n` 后会自动渲染 `languageSelect` slot（位于导航栏右上角），无需手动接入 UI。如需定制下拉项的展示文本，调整每个语言包里的 `displayName` 即可。

如需自定义切换器外观，可在 `DocsLayout` 的 `slots.languageSelect` 中覆盖默认实现。

## 新增语言

1. 在 `src/lib/i18n.ts` 的 `languages` 数组中追加新语言
2. 在 `src/dictionaries/` 下新建 `{lang}.ts`，复制现有语言包并翻译所有值（注意保持 key 一致）
3. 在 `src/dictionaries/index.ts` 的 `dictionaries` 与 `locales` 中注册新语言
4. 在 `content/docs/` 下新建对应语言目录，按章节翻译 Markdown 与 `meta.json`

## 静态导出

由于项目使用 `output: 'export'` 静态导出，不支持 Next.js middleware。根路径 `/` 的语言重定向由根 `page.tsx` 通过客户端重定向完成：

```tsx
// src/app/page.tsx
'use client';
// 通过 window.location.replace 跳转到 /{defaultLocale}
```

> 更多fumadocs UI 翻译配置请参考 `src/lib/layout.shared.tsx`。
