---
title: i18n 国际化
description: Neoverse-Doc 的多语言架构与文案管理方案
---

Neoverse-Doc 基于 fumadocs 内建 i18n 体系，搭配字典化 UI 文案与目录化文档内容，零硬编码即可支持多语言切换。当前已正式启用中文与英文两套界面与文档。

## 架构概览

```text
src/
  dictionaries/
    zh.ts          ← 中文语言包
    en.ts          ← 英文语言包
    index.ts       ← 字典聚合 + getDictionary(locale)
  lib/
    source.ts      ← loader 接入 i18n: { defaultLanguage, languages, parser: 'dir' }
  app/
    page.tsx       ← 根路径 / 客户端重定向到 /{defaultLocale}
    [lang]/
      layout.tsx   ← RootProvider 注入字典 + generateMetadata 按 locale 生成
      page.tsx     ← 首页（lang 来自路由参数）
      docs/
        layout.tsx ← DocsLayout，tree={source.pageTree[lang]}
        [...slug]/page.tsx
      guestbook/
        page.tsx
content/docs/
  zh/              ← 中文文档子树
  en/              ← 英文文档子树
```

> 静态导出 (`output: 'export'`) 不支持 Next.js middleware，所以根路径的语言重定向由根 `page.tsx` 通过客户端 `<meta http-equiv="refresh">` + `window.location.replace` 完成。

## 语言包结构

每个语言包 (`src/dictionaries/{lang}.ts`) 由两部分组成：

- **fumadocs UI 覆盖**：覆盖 `defaultTranslations` 中的搜索、主题、导航等内置文案
- **应用自定义文案**：首页 tagline、导航标签、社区探讨等项目特有文案

```ts
import { defaultTranslations } from 'fumadocs-ui/i18n';

export const zh = {
  ...defaultTranslations,
  displayName: '中文',
  search: '搜索',
  // ...
  // 应用文案
  tagline: '下一代科技美学全栈文档引擎',
  communityTitle: '社区探讨',
} as const;
```

聚合层 `src/dictionaries/index.ts` 暴露 `Locale`、`locales`、`defaultLocale` 与 `getDictionary(locale)` 工具函数。

## 使用方式

**服务端组件（页面 / 布局）**：根据路由 `lang` 参数取字典

```tsx
import { getDictionary, type Locale, defaultLocale } from '@/dictionaries';

export default function Page({ params }: { params: { lang: Locale } }) {
  const dict = getDictionary(params.lang ?? defaultLocale);
  return <h1>{dict.communityTitle}</h1>;
}
```

**客户端组件**：通过 fumadocs 的 `useTranslations()` Hook 访问

```tsx
'use client';
import { useTranslations } from 'fumadocs-ui/contexts/i18n';

export function MyComponent() {
  const t = useTranslations();
  return <span>{t.communityTitle}</span>;
}
```

## 内容文档的多语言

fumadocs-core 的 loader 通过 `parser: 'dir'` 启用目录化解析，按语言子目录组织文档：

```text
content/docs/
  zh/
    getting-started/
    advanced/
  en/
    getting-started/
    advanced/
```

每种语言的 `meta.json` 独立维护页面标题与排序：

```json
// content/docs/en/getting-started/meta.json
{ "title": "Getting Started", "pages": ["index", "syntax-example"] }
```

页面组件通过 `source.getPage(slug, locale)` 按语言取页面：

```tsx
const page = source.getPage(slug, locale);
```

侧栏树 (`source.pageTree`) 为 `Record<lang, Root>`，需按 lang 取：

```tsx
<DocsLayout tree={source.pageTree[lang]} {...options} />
```

## 语言切换器

fumadocs-ui 在检测到 loader 配置了 `i18n` 后会自动渲染 `languageSelect` slot（位于导航栏右上角），无需手动接入 UI。如需定制下拉项的展示文本，调整每个语言包里的 `displayName` 即可。

如需自定义切换器外观，可在 `DocsLayout` 的 `slots.languageSelect` 中覆盖默认实现（参考 fumadocs-ui 源码 `BaseSlots.languageSelect` 类型）。

## 新增语言

1. 在 `src/dictionaries/` 下新建 `{lang}.ts`，复制现有语言包并翻译所有值（注意保持 key 一致）
2. 在 `src/dictionaries/index.ts` 的 `dictionaries` 与 `locales` 中注册新语言
3. 在 `source.config.ts` 与 `src/lib/source.ts` 的 `i18n.languages` 中追加新 locale
4. 在 `content/docs/` 下新建对应语言目录，按章节翻译 Markdown 与 `meta.json`
5. 如需在 Giscus 评论区使用对应语言，更新 `src/components/guestbook.tsx` 的 `GISCUS_LANG_MAP`
